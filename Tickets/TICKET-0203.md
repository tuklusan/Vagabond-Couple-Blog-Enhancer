# TICKET-0203: [nodes.py] Title writer prompt argued with its own deterministic check ("Overland" hard-coded in the default format)
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-05
Description: Observed live (alaska3v1 end-to-end run): step1_title escalated after 6 rounds and the auto-operator aborted the run. Root cause: TICKET-0176 added a deterministic check rejecting 'Overland' on sea journeys AND a prompt sentence saying to use a mode-accurate phrase -- but the SAME system prompt still opened with "Default format: '[Origin] to [Destination] Overland via ...'" with 'Overland' hard-coded. The weak writer (DeepSeek fallback under OpenRouter rate-limiting) anchored to the template word every round; the reviewer prompt ALSO hard-coded 'Overland' in its required-format clause, so even a correct 'Cruise & Road Trip' title risked a REVISE. Self-contradictory instructions with a deterministic check as referee = guaranteed escalation on every mixed-mode post.
Steps to Reproduce: --full run on the alaska cruise post (method 'cruise and drive'); step1_title rounds 1-6 alternate between "', AK' appears 2x" and "'Overland' but journey method is 'cruise and drive'" findings until max_rounds.
Notes: Fixed by removing the ambiguity entirely: new `_journey_mode_phrase(method)` picks the journey word IN CODE (sea+land -> 'Cruise & Road Trip'; sea only -> 'Cruise'; else 'Overland') and both the writer's default-format sentence and the reviewer's required-format clause are built with that phrase ("Use EXACTLY the journey phrase '...' -- do not substitute another travel word"), so prompt, reviewer, and deterministic check all agree by construction. Also sharpened the suffix rule the round-1 failure exposed: a 2-letter state/province code may appear AT MOST ONCE in the whole title (the old wording only covered waypoints sharing a state, not destination+waypoint duplication like 'Fairbanks, AK ... Ketchikan, AK'). Tests: test_journey_mode_phrase (mode selection for mixed/sea/land/unknown + proof the chosen phrase passes the 0176 deterministic check). Run resumed via --resume with the 1J audit artifact (423 verdicts + certifications) fully preserved.
