# TICKET-0172: [sequencer.py] Multi-instance phrase repetition unfixable -- single-passage remediation cannot converge on an N-instance running gag
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-04
Description: Observed live (alaska2v1): the original author's caption gag "Yes, we took the photo. No, we can't control the weather." appears in 9 captions. Per the operator's standing decision (see memory: source repetition IS a defect to fix, 2026-07-04), this must be remediated -- but every existing path fixes at most ONE passage per attempt: Step 12's writer rewrite targets the single best-matching node, and the Phase 5 bounce loop allows MAX_PASS1_BOUNCES=3 single-fix attempts. Nine instances can never converge; the run halts for the operator, defeating full autonomy.
Steps to Reproduce: Any post with the same sentence repeated across >3 captions/paragraphs; Phase 5 Pass-1 flags REPETITION; watch the bounce loop exhaust without fixing it.
Notes: Fixed with deterministic condensation -- `_dedupe_html(html, phrase)`: keep the FIRST occurrence of the flagged phrase, remove the sentence(s) carrying it from every later body text node. Matching is normalized (case/punctuation/apostrophe-variant-proof); a sentence is removed when its normalized text contains the phrase OR is a substantial (>=10 char) fragment of it (handles a two-sentence gag where the deterministic 1H scan only keyed on one half). Edits touch ONLY the NavigableString holding the sentence -- markup and every <a href> survive verbatim (G3), other sentences in the same caption/paragraph are kept. Wired in at BOTH stages: (1) `step12_resolve_node` tries dedupe first for kind=repetition items (fixing duplicates pre-assembly, where it belongs); (2) the Phase 5 bounce loop extracts quoted phrases from the Pass-1 REVISE (revision_instructions + repetition findings) and condenses any phrase occurring in >1 node BEFORE considering a factoid drop or single-passage rewrite -- targeting the reviewer's actual complaint before sacrificing any content. `_dedupe_repeated_phrase` applies the same transform to the saved `pre_assembly_source` so a later bounce's reassembly cannot resurrect the removed duplicates. Tests: test_dedupe_html_condenses_later_instances, test_quoted_phrases_extraction. Known conservative limitation (accepted): an instance whose entire node text IS the phrase, or whose sentence spans multiple inline tags, is left intact rather than emptied/mangled -- residual repetition of that shape still ends in an operator halt rather than a bad edit.
