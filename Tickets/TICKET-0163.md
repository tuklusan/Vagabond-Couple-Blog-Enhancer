# TICKET-0163: [sequencer.py/nodes.py] Auto-remediate Pass-1 repetition/whiplash instead of only accepting or halting
Status: Closed
Priority: Medium
Type: Feature
Created: 2026-07-04
Description: The Phase 5 G2 Pass-1 holistic reviewer (the "18-year-old reader" persona, TICKET-0161) can flag REPETITION or SMOOTH_READ (timeline/route "whiplash") findings that are NOT localizable to a pipeline-generated factoid. Previously the only bounce path (`_drop_flagged_factoid`) handled factoid-rooted findings; anything else fell straight through to an operator halt, and prior triage in this project had (incorrectly) treated repetition confirmed-present-in-the-original-source as an accepted, out-of-scope stopping point rather than a defect to fix. The user explicitly corrected this: repetition/whiplash is a real defect regardless of which pass introduced the wording, and since the pipeline is meant to run fully autonomously, it should have a real remediation path rather than always stopping for a human. | Suggestion: add a passage-localization + rewrite step reusing the existing (previously dormant/unwired) `step12_resolve` writer<->reviewer node.
Steps to Reproduce: Run Phase 5 on a post where Pass-1 flags a REPETITION or SMOOTH_READ finding rooted in body prose (not a Step 9-F factoid); observe the run previously fell through to an operator halt with no attempt at an automated fix.
Notes: Implemented via:
- `validators.body_content_tags(html, tag_names)` (orchestrator/validators.py) -- generalized the existing `body_h2_tags` post-`<!--more-->` walk (TICKET-0154/0158) to accept arbitrary tag names, reused for passage lookup. `body_h2_tags` now delegates to it.
- `sequencer._locate_flagged_passage(html, needle)` -- token-overlap search (same scoring approach as the existing factoid-bounce localizer, TICKET-0123) over body `<p>`/`<td>`/`<li>` tags to find the single best-matching passage for a Pass-1 finding, requiring a minimum overlap score of 2 to avoid false-positive matches.
- `sequencer._remediate_flagged_passage(sctx, pass1)` -- when `_drop_flagged_factoid` can't localize the REVISE, this localizes the actual passage, invokes `node_specs.step12_resolve()` via the standard writer<->reviewer loop (with the passage + issue text as context), verifies the rewrite preserves every original `<a href>` in the passage byte-for-byte (G3; rejects the fix and falls through to halt if a link was dropped), splices the certified replacement into the live working-HTML tree via `assembler._frag()` + `Tag.replace_with()`, and re-certifies. Wired into `phase5_certification_node`'s existing `MAX_PASS1_BOUNCES` loop, after the factoid-drop attempt.
- `nodes.step12_resolve()`'s writer/reviewer prompts (orchestrator/nodes.py) extended to explicitly cover WHIPLASH (a passage jumping back to an earlier waypoint/timeline point after the narrative moved on) alongside repetition/rules violations, and to require every existing `<a href>` in the passage survive verbatim.
- New tests in tests/test_sequencer.py: `test_locate_flagged_passage_finds_best_match`, `test_locate_flagged_passage_no_match_returns_none`, `test_remediate_flagged_passage_rewrites_and_preserves_hrefs`, `test_remediate_flagged_passage_rejects_dropped_href` (writer/reviewer mocked, no live API calls).
Follow-up noted but NOT fixed here (separate, smaller scope): `step12_resolve` is also wired at Phase 3 (pre-generation) via a plain (non-iterating) `g(...)` node with no items_fn, so `context["issue"]/context["passage"]` are always empty there and that pre-generation call is presently a no-op that always skips cleanly. Worth converting to an iterating node driven by real 1H/1I findings in a future ticket.
