# TICKET-0195: [vision_client.py] Unsafe string formatting in _CERTIFY_PROMPT can crash the scr
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-05
Description: The prompt template uses old-style % formatting with user-provided original and proposed text. If either contains a '%' character, it will raise a ValueError (e.g., 'not enough arguments for format string' or 'unsupported format character'), causing the script to crash. This happens in certify_correction and is not caught, potentially halting the audit pipeline. | Suggestion: Escape percent signs in the original and proposed strings before interpolation, e.g., original_escaped = (original or '(none)').replace('%', '%%'), and similarly for proposed. Or replace the %-based template with a manual concatenation/join that does not interpret format specifiers. | File: orchestrator/vision_client.py | Severity: critical
