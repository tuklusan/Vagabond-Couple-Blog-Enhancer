# TICKET-0204: [image_audit.py] Certified caption corrections flatten the author's voice
Status: Closed
Priority: Low
Type: Enhancement
Created: 2026-07-05
Description: Observed in the alaska3v1 end-to-end run (the first with detection + second-VLM certification + application all active): the certified corrections are factually excellent -- 12-sample grading found 10 clear improvements, 0 nonsense, 0 fabrication, coordinates/places retained throughout, including reading 'WHITEPASS / SKAGWAY 20.4 MILES' off a sign and correcting 'Elk' to 'Moose' -- but ~2/12 replaced the author's humorous captions ('looking suspiciously sunny for a place that measures rainfall in feet') with flat literal scene descriptions. Accurate, certified, and personality-free: a stylistic regression for reader-visible captions even when factually superior.
Steps to Reproduce: Output/runs/alaska3v1/artifacts/1J_image_audit.json, applied caption findings for the Ketchikan waterfront / Herring Cove images.
Notes: IMPLEMENTED (prompt-level): the 1J audit prompt now instructs 'PRESERVE THE AUTHOR'S VOICE: if the original text has humor, wordplay, or a personal tone that is still compatible with what the frame shows, carry that tone into the correction instead of replacing it with a flat literal description -- fix only the factual mismatch, not the personality.' The second-VLM certifier's natural-prose rule (TICKET-0175 rule 2) still guards against the tone-carrying rewrite becoming inaccurate or awkward. Not re-validated live yet -- deliberately: exercising it costs a full re-audit (424 detection + certification calls), so validation happens organically on the NEXT vision-enabled run the user requests (the vision audit is opt-in only per TICKET-0202). If voice-flattening persists on that run, the follow-up is to feed the certifier an explicit tone-comparison criterion rather than relying on the proposer's prompt alone.
