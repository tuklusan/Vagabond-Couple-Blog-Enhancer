# TICKET-0208: [research_client.py/nodes.py/sequencer.py] Vision-nudged, research-grounded separators -- retrieval from real libraries/academic sources with evidence-entailment review
Status: Closed
Priority: High
Type: Feature
Created: 2026-07-05
Description: User directive (2026-07-05, resolving the peru-2023 193-pair separator impasse): "use vision-nudged, context-driven, deeper search to produce verifiable information from actual online libraries and academic research papers." The impasse: rev-18 Step 13 demands factually-grounded separators and assumes web research, but the app's reviewer is permanently web-less (DeepSeek-only), so it rejected most world-knowledge claims and 193 pairs could never converge.
Steps to Reproduce: peru1v1 step13 pre-fix: mass ESCALATEs (51 writer calls over 29 pairs, mostly unverifiable-fact rejections).
Notes: Architecture -- turn fact VERIFICATION (needs the web) into evidence ENTAILMENT (needs only reading):
1. `research_client.py` (new): keyless retrieval from Wikipedia REST (search -> page summaries) and OpenAlex (academic works; abstracts reconstructed from the inverted index), both verified live ('Inca Architecture and Construction at Ollantaytambo', 1996, with abstract). All fetches through the SSRF-safe redirect-validating safe_get with the project User-Agent; every failure degrades to fewer snippets, never an exception. `research(queries)` dedupes by title; `format_evidence()` renders the numbered packet.
2. Vision nudge (opt-in, `ORCH_STEP13_VISION=1`, same explicit-per-run rule as the 1J audit -- authorized by the user for this peru run): `vision_client.inspect_image_pair()` sends BOTH adjacent photos in one call (multi-image support added to _inspect_with_models) and returns searchable subject phrases + a per-photo visible description, logged to the transcript.
3. Context-driven queries: visual subjects (or the captions when vision is off) anchored with the destination country/region; enrichment lives in `sequencer._enrich_separator_item`, attached per pair item.
4. `step13_separator` prompts rebuilt around the packet: the WRITER may state only facts directly supported by the numbered snippets (plus plain descriptions of the stated visible content) -- no memory facts; the REVIEWER's FACTS criterion becomes EVIDENCE ENTAILMENT against the same packet ("FAIL any claim the snippets do not support, even if you believe it is true"), with REVISE-toward-the-supported-facts preferred over ESCALATE. Temporal rule (TICKET-0207) applies to both. With no evidence retrieved, both prompts fall back to the previous guidebook-fact behavior.
Tests: tests/test_research.py (16 checks: abstract reconstruction, packet formatting, dedup, writer/reviewer evidence modes + fallback, pair-item enrichment with mocked VLM+research). Live smoke: Wikipedia + OpenAlex returned real Ollantaytambo scholarship; prompt threading verified with live snippets.
