# TICKET-0175: [image_audit.py] 1J is an excellent detector but an unreliable corrector -- corrections need visual certification before production
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-04
Description: Human review of the alaska2v1 full run (430 findings, 318 applied corrections across alt/title/caption) graded a 21-correction sample: ~50% genuine improvements (e.g. "Ketchikan Post Office" corrected to "Frontier Shipping Express" by READING THE VISIBLE SIGN; a whale-feeding factoid removed from a train photo's caption; "child asleep at table" corrected to the two adults actually pictured), ~15% harmless no-op noise, ~30% information-degrading (place names/facts dropped: "Glacier Gardens' gift shop..." replaced by bland scene description; "MS Statendam, Hecate Strait" dropped from a title), and ~2 catastrophic grammatical nonsense ("A body of water entered Alaska cruise service in 2011 after extensive renovations" -- the VLM substituted its generic scene noun into the original sentence structure). Extrapolated over 318 applied corrections: ~6 nonsense, ~90 degrading. The model also violated its own prompt rules in some flags (denying proper nouns it cannot verify: "the image does not show Herring Cove"; denying vantage claims), which no deterministic gate can catch.
Steps to Reproduce: See Output/runs/alaska2v1/artifacts/1J_image_audit.json findings with applied=true; grade a random sample.
Notes: Design (to implement before 1J corrections can be trusted unsupervised):
1. **Visual certification loop (the project's own writer<->reviewer pattern, applied to vision):** every proposed correction goes back to a SECOND VLM call (different model from the chain when possible) with the image + corrected text: certify (a) the corrected text accurately describes the visible content, (b) it is grammatical, natural prose (explicitly reject generic-noun substitution artifacts), (c) it does not deny/replace any proper noun, coordinate, or vantage claim that pixels cannot disprove. REVISE loops like Tier-1; uncertifiable -> finding-only, never applied.
2. **Extend the retention gate to alt/title** for place/entity words when the location is not the contradiction (currently captions only; observed titles losing "MS Statendam, Hecate Strait").
3. **No-op suppression:** reject corrections whose normalized text equals the original (observed: punctuation-only "corrections").
4. **Rule-violation rejection:** reject a correction whose reason amounts to denying an unverifiable specific (patterns like "does not show <ProperNoun>" where the visible description is compatible with the claim).
Until this lands, production posture should be: alt/title/caption corrections DISABLED by default (findings-only artifact + Phase 4 summary for operator review), enabled via ORCH_IMAGE_AUDIT_APPLY=1 for supervised runs.
Resolution: IMPLEMENTED (2026-07-04). (1) `vision_client.certify_correction`: every proposed correction gets a second-opinion VLM call on a ROTATED model chain (certifier != primary proposer when >1 model up) judging accuracy / natural prose (explicitly rejecting generic-noun substitution artifacts) / no unjustified proper-noun-or-vantage deletion; fails CLOSED (no verdict -> not approved). Certification happens at audit time while image bytes are in hand; the result is cached in the per-image verdict (resume never re-pays) and logged to ai_transcript.txt. Only status=CONTRADICTED + gates-passed + CERTIFIED proposals enter the corrections list; everything else stays findings-only with the rejection reason recorded. (2) No-op suppression added to _acceptable_correction (normalized case/punctuation equality). (3) Alt/title proper-noun retention delegated to the certifier's rule 3 (deterministic retention stays hard for captions). Live smoke (14 alaska images, fresh audit): 15 contradiction flags -> only 3 corrections applied, all three verifiably sound (one read 'STATENDAM ROTTERDAM' off the hull); the certifier rejected a proposal that itself hallucinated a 'cargo ship with cranes', and rejected every ship-omitting rewrite by identifying the ship in-frame. The trade is deliberately fail-closed: some valid corrections get rejected (finding-only costs nothing), no observed unsound correction survived. Tests: test_certifier_rejection_demotes_to_finding_only, test_certifier_approval_allows_correction, test_noop_correction_suppressed. ORCH_IMAGE_AUDIT_APPLY remains default-off; with certification landed, enabling it on the next supervised run is the intended validation step.
