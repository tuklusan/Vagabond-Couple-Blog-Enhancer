# TICKET-0168: [image_audit.py] Unhandled exceptions in vision client calls
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-04
Description: `vision_client.fetch_image` and `vision_client.inspect_image` are called without try/except. If either raises an exception (network error, invalid response, disk I/O, etc.), the entire `audit_images` loop terminates, losing all progress beyond that point. The function is designed to be resilient with caching, but one failed image can break the whole pipeline. | Suggestion: Wrap the `vision_client.fetch_image` and `vision_client.inspect_image` calls in a `try-except` block, log the error, increment a failure counter, and continue to the next image. Consider adding a `max_consecutive_failures` guard to abort if too many fail. | File: orchestrator/image_audit.py | Severity: critical

Notes: PARTIALLY VALID, FIXED. Triage: the "break the whole pipeline" claim is overstated -- sequencer.image_audit_node() already wraps audit_images in try/except and degrades to status=unavailable, so a raise could never crash the run. But the per-image resilience claim is real: inspect_image had an uncaught raise path (a 200 response with a malformed/non-JSON body raises KeyError/ValueError out of `resp.json()["choices"][0]["message"]`), and one such image would abandon the REST of that audit run (progress cache limits the loss to a resume, but mid-run abandonment contradicts the module's own design). Fixed per the reviewer's suggestion, matching the TICKET-0143 per-item precedent: (1) vision_client.inspect_image now catches ValueError/KeyError/IndexError/TypeError around response extraction and fails over to the next model in the chain; (2) image_audit.audit_images wraps the per-image fetch+inspect in try/except -- counts a review_failure, logs, continues; (3) added MAX_CONSECUTIVE_FAILURES=5 early-stop (also the reviewer's suggestion): a streak of failures means the provider/key/network is down for the run, so stop burning full retry chains on every remaining image; scattered failures reset the streak, and the progress artifact lets the next run resume where it stopped. New tests: test_audit_survives_per_image_exception, test_audit_stops_after_consecutive_failures (both pass; full image-audit suite 31 checks green).
