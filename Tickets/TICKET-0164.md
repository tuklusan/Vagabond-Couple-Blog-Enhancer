# TICKET-0164: [sequencer.py] Wire step12_resolve to real 1H/1I findings (was a silent no-op)
Status: Closed
Priority: Medium
Type: Bug
Created: 2026-07-04
Description: The Phase 3 `step12_resolve` node was registered via the plain (non-iterating) `generative_node`, which runs it exactly once with `sctx.context` untouched. But `nodes.step12_resolve()`'s writer/reviewer prompts read `context.get("issue", "")` and `context.get("passage", "")` -- keys nothing in the sequence ever populated. So this node has silently built an empty "Flagged issue: \nPassage to fix:\n\n..." prompt on every run since it was added, and the real 1H_repetition/1I_writing_rules findings (which DO run earlier and produce real data) were never fed into it. Being `optional=True`, it just skipped cleanly every time with no visible symptom. Found while investigating TICKET-0163 (Pass-1 repetition/whiplash auto-remediation).
Steps to Reproduce: Run the full sequence on any post with real 1H/1I findings (repeated sentences, non-curated forbidden words, first-person narrator slips); observe `gen_step12_resolve` was always effectively empty/skipped regardless of what 1H/1I actually found.
Notes: Fixed by:
- `sequencer._repetition_rule_items(sctx)` -- reads `analysis_1H_repetition` / `analysis_1I_writing_rules` artifacts and builds one item per real finding: a repeated sentence, a forbidden word/phrase NOT already covered by the deterministic curated-synonym swap (`assembler._FORBIDDEN_SYNONYMS`, applied later at Phase 5 assembly), or a first-person narrator sentence (max 3). Each item carries a `needle` (fed to `_locate_flagged_passage`, from TICKET-0163) and an `issue` description.
- `sequencer.step12_resolve_node()` -- a custom node (not the generic `iterating_generative_node`, since each fix must splice into the LIVE working-HTML tree immediately so the next item's passage is located post-splice, not against a stale copy) that locates each item's passage, runs the writer<->reviewer loop, and splices certified fixes in place via the shared `_splice_passage_fix` helper. Per-item exceptions are caught and recorded as skips rather than aborting the whole node (same precedent as TICKET-0143 for the other optional per-item nodes). Never halts the run.
- `sequencer._splice_passage_fix(sctx, node, new_html)` -- extracted from `_remediate_flagged_passage` (TICKET-0163) so both the Pass-1 bounce path and this Step-12 node share one splice implementation (href-diff -> replace_with -> re-serialize from the root) instead of duplicating it.
- `build_full_sequence()` now uses `step12_resolve_node()` in place of the old no-op `g("step12_resolve", ..., optional=True)` registration.
- New tests in tests/test_sequencer.py: `test_repetition_rule_items_reads_1H_1I_findings`, `test_step12_resolve_node_splices_each_fix_and_records_skips` (writer/reviewer mocked).
Full deterministic suite (validators/context/assembler/document_cert/sequencer) plus test_full_sequence.py dry walk all pass.
