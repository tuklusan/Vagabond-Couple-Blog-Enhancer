# TICKET-0128: Schema forced 'USA' onto non-US places + hardcoded touristType=Overlander + generic instrument fallback
Status: closed
Priority: High
Type: Bug
Created: 2026-07-03
Description: schema_builder._full_place_name() unconditionally appended ', USA' to any place name lacking a country mention, producing 'Kyiv, Ukraine, USA' in the arsenalna1 run's TravelAction schema (fromLocation/toLocation) -- factually wrong, bad for schema.org validity/SEO. Root cause: the function was written assuming every post is a US road trip. Also touristType was hardcoded to the literal string 'Overlander' regardless of post type (POSTED uses 'Urban sightseer' for this transit/descent post), and _instrument() fell back to the generic 'Overland vehicle' even when context['vehicle'] was empty (no personal vehicle in this post at all -- it's a metro descent). FIX: (1) _full_place_name only appends USA when a real 2-letter US state code was actually expanded; (2) added _tourist_type(context) -- 'Overlander' only when context['vehicle'] is truthy, else neutral 'Traveler' (not a fabricated classification, but no longer factually implies a road trip); (3) _instrument() falls back to the extracted context['method'] (e.g. 'escalator') before the generic 'Overland vehicle' filler. NOTE: matching POSTED's more specific 'Urban sightseer' classification and 'Kyiv Metro' instrument name would need deeper post-type inference -- deferred as a future enhancement, not attempted here to avoid another guess-based fabrication. Verify: re-run arsenalna1, confirm schema fromLocation/toLocation say 'Kyiv, Ukraine' (no USA), touristType='Traveler', instrument name derived from method not a car brand.
