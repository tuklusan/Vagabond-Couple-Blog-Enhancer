# TICKET-0171: [sequencer.py] Pass-1 factoid-drop mislocalizes on section-title tokens, destroying innocent content
Status: Closed
Priority: High
Type: Bug
Created: 2026-07-04
Description: Observed live (alaska2v1 full run): Phase 5 Pass-1 flagged a repeated CAPTION phrase ("Yes, we took the photo. No, we can't control the weather", 9 instances in the original author's captions). `_drop_flagged_factoid` scored factoid SECTION TITLES against the revision text -- the complaint mentioned "Ketchikan"/"Vancouver"/"weather", which token-matched three factoid section titles -- so all 3 bounces were spent dropping certified-good factoids that never contained the flagged phrase, and the run still halted. Worse, the drop DESTROYED the factoid output (`item["output"] = ""`), losing certified content with no recovery path in the artifact item (recovered manually this run from the parallel `outputs` list).
Steps to Reproduce: Run Phase 5 cert on a post whose Pass-1 REVISE quotes a caption phrase while mentioning place names that appear in factoid section titles.
Notes: Fixed with an evidence gate: a factoid is dropped only when (a) a QUOTED flagged phrase (extracted via `_quoted_phrases`, normalized apostrophe/case/punctuation-proof via `_normalize_text`) actually appears in the factoid's OUTPUT text, or (b) the finding tokens overlap the factoid's OUTPUT (not merely its title) with combined score >= 4. And the drop now preserves the removed text in `dropped_output` instead of destroying it. New tests: test_drop_factoid_requires_output_evidence (both directions: no drop without output evidence; drop + preservation with evidence). See also [[TICKET-0172]] (the mechanism that now handles what this complaint actually was) and TICKET-0123 (the original bounce localizer this hardens).
